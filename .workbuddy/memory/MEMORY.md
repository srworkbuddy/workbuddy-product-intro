# WorkBuddy产品介绍项目记忆

## 项目概述
整理WorkBuddy产品介绍材料，包括场景实践、使用案例等，生成PPT/DOCX/HTML三种格式

## 关键技术方案
- **DOCX生成**: 使用python-docx (系统Python /usr/bin/python3)，避免JS中文引号问题
- **PPTX生成**: 使用pptxgenjs直接API（非html2pptx浏览器截图方案）
  - 形状类型引用: `pptx.shapes.RECTANGLE` / `pptx.shapes.ROUNDED_RECTANGLE` / `pptx.shapes.OVAL`（不是`pptx.ShapeType.*`）
- **HTML交互版**: 独立完整，含9大业务域分类场景卡片+4个案例+21条素材索引

## 踩坑记录
- JS中中文引号（""）会导致语法错误 → 改用Python生成DOCX
- python-docx的lxml在沙箱Python中dlopen失败 → 用系统Python
- html2pptx浏览器方案权限受限且慢 → 改用pptxgenjs直接API
- pptxgenjs的ShapeType不是pptx.ShapeType → 应该用pptx.shapes.RECTANGLE等
- 场景卡片从上下布局改左右布局时，需用JS将现有子元素包裹进.scene-content再插入.scene-preview
- 乐享内网页面图片路径为相对路径 `/assets/xxx`，需加域名前缀 `https://csig.lexiangla.com/assets/xxx` 才能在HTML中引用
- 乐享内网页面需登录认证，WebFetch无法直接抓取；需用MCP工具 entry_describe_ai_parse_content 获取内容
- 内网图片在公网环境可能无法加载，建议用公开文章图片作fallback（onerror属性）
